# LTS ナレッジベース ルールブック

## プロジェクト概要

株式会社ライフタイムサポートの会社ルール・AI開発フロー・権限設計などを記録した
社内ナレッジベース。全 AI（Claude Code 等）がこのリポジトリを参照して動作する。

## ファイル構成

```
lts-knowledge/
├── 01_company_mindset.md     ← 会社の価値観・経営思想
├── 02_folder_structure.md    ← Gドライブ・Gitの統一フォルダ構成
├── 03_permission_rules.md    ← 情報公開範囲（①全社 ②チーム ③経営陣 ④龍竹専用）
├── 04_ai_development_flow.md ← AI を使った開発・自動化の標準フロー
├── 05_client_repo_rules.md   ← クライアントリポジトリの命名規則・運用ルール
└── handoff_YYYY-MM-DD.md     ← セッション引き継ぎメモ（日付付き）
```

## 作業ルール

### 内容の追加・更新
- 新しいルール・知見を追加する前に、既存ファイルに統合できないか確認する
- ファイルを新規作成する場合は `番号_トピック名.md` の命名規則に従う
- 過去の引き継ぎメモは `handoff_YYYY-MM-DD.md` 形式で保存する（上書きしない）

### Pull Request 運用
- AI（Claude Code）が自発的に改善提案する場合は PR を作成し、龍竹が承認してからマージする
- 緊急の修正（誤情報の削除等）は直接コミットして構わない

### このリポジトリを参照するタイミング
- 会社のルール・フォルダ構成・権限に疑問が生じた時
- グローバル CLAUDE.md（`~/.claude/CLAUDE.md`）に記載のない事項の判断が必要な時
- 新しいプロジェクトを開始する前の確認

## 重要原則

**憶測で動かない。** 判断に迷ったら必ずこのリポジトリの内容を参照し、
それでも不明な場合はユーザーに確認する。

## クレデンシャル管理ルール

**トークン・APIキーを取得/受領した場合は、必ず `lts-knowledge/06_credentials.md` に記録すること。**

- 記録内容: サービス名・用途・キーの値・取得日・有効期限
- トークンやAPIキーが必要になった場合は、まず `lts-knowledge/06_credentials.md` を確認してから質問する
