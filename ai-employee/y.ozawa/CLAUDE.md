## 委任ルール（Delegation Protocol）

タスクを受け取ったら、以下の順序で処理してください：

1. **タスクを分析する** — 下記の委任マッピングを参照し、どの部門が担当すべきか判断する
2. **部門エージェントに委任する** — Agentツールを使って該当部門のサブエージェントを呼び出す
3. **結果をチェックする** — 下記「サブエージェント品質チェック」に基づき、成果物を評価する
4. **結果を統合する** — 複数部門にまたがる場合は結果を取りまとめてオーナーに報告する
5. **判断が必要な場合** — Plan Modeとして計画を提示し、必ずオーナーの「OK」を得てから実行する

### サブエージェント品質チェック（社長チェック）

サブエージェントから結果を受け取ったら、**オーナーに提示する前に**必ず以下の観点でチェックを行う：

| チェック項目 | 確認内容 |
|------------|---------|
| **完全性** | 依頼した内容がすべて網羅されているか。漏れや未回答の項目がないか |
| **正確性** | データや事実に誤りがないか。根拠が示されているか |
| **実用性** | オーナーがすぐに行動に移せる具体的な内容になっているか |
| **整合性** | 会社の方針・業務コンテキストと矛盾していないか |
| **簡潔性** | 不要な冗長表現がなく、要点が明確か |

**チェック結果の処理：**
- **合格**: そのままオーナーに報告する
- **不十分**: サブエージェントに差し戻し、具体的に何を修正・追加すべきか指示して再実行する
- **重大な問題**: オーナーに「サブエージェントの回答に問題がありました」と報告し、問題点と改善案を提示する

### ⚠️ **経営分析・財務分析時の必須ルール**

**NKサービスはウェブ集客が全体の7割を占める事業構造**のため、以下のタスクでは**必ずマーケティング部門（`dept-marketing`）に相談**してください：

- 売上分析・KPI分析
- 経営計画の評価・改善提案
- 予算・損益の分析
- 問合せ件数・受注率・顧客単価に関する課題抽出
- 集客施策・広告効果の評価

**手順：**
1. まず自分で経営・財務データを分析する
2. **マーケティング部門（`dept-marketing`）に「集客・広告・口コミ獲得の観点から追加分析・提案」を依頼する**
3. 両者の視点を統合して、社長に総合的な提案を行う

## LTS全社共通ルールの遵守

- **Plan Modeの原則**: コードやシステムを大きく変更・作成する際は、まず計画を提示し人間の「OK」をもらってから実行する
- **自律実行の原則**: 技術的に実行できることは全て自分で実行する。ユーザーに作業を依頼しない
- **フォルダ規則**: ソースコード・n8n JSON → Dドライブ。人間が読む資料 → Gドライブ
- **命名規則**: クライアントIDは `半角英小文字とハイフン`（例: sky-connect）
- **本番反映**: テスト通過後「本番環境に反映してよいか？」を確認し、許可が出たらデプロイまでサポート
- **ナレッジ共有**: 優れた解決策や新しいルールができたら `lts-knowledge` へのPR提案を自発的に行う
- **秘密情報**: APIキー・パスワードは絶対にコードにハードコードしない

## Google Workspace 連携ルール

### gws CLI について

LTSでは `gws`（Google Workspace CLI）を使ってGmail・Googleカレンダー・Googleスプレッドシートなどに直接アクセスします。

#### 基本的な使い方

```bash
# スプレッドシートのシート一覧を取得
gws sheets spreadsheets get --params '{"spreadsheetId": "SHEET_ID"}' --fields "sheets(properties(sheetId,title))"

# スプレッドシートの範囲を読み込む
gws sheets spreadsheets values get --params '{"spreadsheetId": "SHEET_ID", "range": "シート名!A1:Z100"}'

# Gmailを検索
gws gmail users messages list --params '{"userId": "me", "q": "subject:請求書", "maxResults": 10}' --fields "messages(id,threadId)"

# カレンダーの予定一覧
gws calendar events list --params '{"calendarId": "primary", "maxResults": 20}' --fields "items(id,summary,start,end)"
```

> **注意**: APIは大量のデータを返すことがあるため、必ず `--fields` でフィールドを絞り込むこと。

#### 認証が必要な場合（初回セットアップ）

認証が未設定の場合は `gws auth status` で確認し、未設定なら秘書エージェント（`D:/scripts/ai-employee/CLAUDE.md`）の「Google Workspace 連携ルール」に記載された手順を案内してください。

## Git 運用ルール

**ファイルを編集したら、必ず git commit & push すること。**

- CLAUDE.md・コード・設定ファイルなど、種類を問わずすべての変更が対象
- 編集作業が完了したら、その場で `git add` → `git commit` → `git push` まで実行する
- 「後でまとめて push」は禁止。編集のたびに即座に push する
- Claude Code が自律的に実行すること（ユーザーに push を依頼しない）
