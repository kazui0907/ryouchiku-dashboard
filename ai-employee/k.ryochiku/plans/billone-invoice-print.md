# ビルワン請求書印刷自動化 — 実装計画

## 概要

当社宛に届くビルワン（Bill One）からの請求書を、一括でダウンロード・印刷できる自動化スキルを構築する。

## 調査結果

### Bill One について
- Sansanが提供するインボイス管理サービス
- 紙・PDF・メールなど様々な形式の請求書を一元管理
- 請求書受領・データ化完了時にメール通知機能あり

### API連携状況
- 公式APIは存在するが、kintone/奉行クラウド等のパートナー向け
- 一般ユーザー向けの公開APIは利用不可（オーナー確認済み）

### 通知メールについて
- Bill Oneから「請求書が届きました」という通知メールがGmailに届く
- メール内にビルワンへのリンクがある（PDF直接ダウンロードリンクの有無は要確認）

---

## アプローチ比較

### アプローチA: Gmail通知メール経由（推奨）

**概要**: Gmailに届くBill One通知メールを検索し、メール内のリンクからPDFを取得

| 評価項目 | 評価 |
|---------|------|
| 実装難易度 | ★★☆☆☆（既存gmail-helper.js活用可） |
| 安定性 | ★★★★☆（メール形式が変わらない限り安定） |
| 保守性 | ★★★★☆（UI変更の影響を受けにくい） |
| 規約リスク | ★★★★★（Gmailは正規API使用） |

**メリット**:
- 既存のgmail-helper.jsを活用できる
- ビルワンへの直接ログインが不要になる可能性
- Gmailをトリガーとしてn8nで自動化可能

**デメリット**:
- 通知メール内のリンクがセッション付きの場合、PDFダウンロードにログインが必要
- メール形式変更時に修正が必要

### アプローチB: ブラウザ自動化（スクレイピング）

**概要**: Puppeteer/Playwrightでビルワンにログインし、PDFを一括ダウンロード

| 評価項目 | 評価 |
|---------|------|
| 実装難易度 | ★★★★☆（ログイン・2FA対応が複雑） |
| 安定性 | ★★☆☆☆（UI変更で壊れやすい） |
| 保守性 | ★★☆☆☆（定期的なメンテナンスが必要） |
| 規約リスク | ★★☆☆☆（スクレイピングは規約違反の可能性） |

**メリット**:
- 確実にPDFをダウンロードできる
- 未印刷の請求書を一覧から選択可能

**デメリット**:
- ビルワンのUI変更で壊れるリスクが高い
- ログイン認証（特に2FA）の維持が複雑
- サービス規約に抵触する可能性

## 推奨プラン: アプローチA（Gmail経由）

### 前提条件の確認（実装前に必要）

1. **ビルワンからの通知メールの内容を確認**
   - メール内にPDF直接ダウンロードリンクがあるか
   - リンクがセッション付き（ログイン必須）かどうか
   - PDFがメールに添付されているか

2. **確認方法**
   ```bash
   # ビルワンからの通知メールを検索
   node gmail-helper.js ryouchiku@life-time-support.com triage --query="from:bill-one.com" --max=5

   # メール内容を確認
   node gmail-helper.js ryouchiku@life-time-support.com get <MESSAGE_ID>
   ```

## 実装計画

### Phase 1: 調査・検証（まず最初に実施）

1. ビルワンからの通知メールを取得して内容を分析
2. PDF取得方法を特定（添付/リンク/ログイン必須）
3. 実現可能な方式を最終決定

### Phase 2: コアロジック実装

#### シナリオA: PDF添付または直接ダウンロードリンクがある場合

```
billone-invoice-helper.js
├── searchBillOneEmails()     # 通知メールを検索
├── extractPdfLinks()         # メールからPDFリンクを抽出
├── downloadPdfs()            # PDFをダウンロード
└── printPdfs()               # 一括印刷（lp/lprintコマンド）
```

#### シナリオB: ビルワンログインが必要な場合

```
billone-invoice-helper.js
├── searchBillOneEmails()     # 通知メールを検索
├── extractInvoiceLinks()     # メールから請求書詳細リンクを抽出
├── loginToBillOne()          # Puppeteerでログイン
├── downloadPdfs()            # PDFをダウンロード
└── printPdfs()               # 一括印刷
```

### Phase 3: スラッシュコマンド化

`/.claude/commands/print-invoices.md` として実装：

```markdown
---
description: ビルワンの請求書を一括印刷
---

# /print-invoices

ビルワンから届いた請求書のPDFを一括でダウンロードし、印刷します。

## 引数
- --days=N : 過去N日以内の請求書を対象（デフォルト: 7）
- --dry-run : 印刷せずにダウンロードのみ

## 実行手順
1. Gmailからビルワン通知メールを検索
2. 各メールからPDFをダウンロード
3. ダウンロードフォルダに保存
4. 一括印刷を実行
```

### Phase 4: 営業アシスタントへの統合

`dept-sales.md` に以下を追加：

```markdown
### 5. 請求書印刷（ビルワン連携）

ビルワンから届いた請求書PDFを一括で印刷。

**使用方法:**
- `/print-invoices` — 過去7日分の請求書を印刷
- `/print-invoices --days=30` — 過去30日分を印刷
- `/print-invoices --dry-run` — 印刷せずリスト確認のみ
```
