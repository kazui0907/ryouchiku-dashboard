## 1. システム概要

### 1.1 目的

メール処理を自動化しつつ、**Claude Code がすべての処理を担当**することで、修正指示がナレッジとして蓄積される仕組みを構築する。

### 1.2 基本方針

- **既存の `/reply-mail` コマンドはそのまま維持**（並行利用可能）
- **メール作成ルール・ラベル振り分けルールは現状を引き継ぐ**
- **ナレッジファイルは既存のものを継続使用**
- **Discord はコミュニケーション手段として追加**（処理の主体は Claude Code）

## 3. 既存資産の引き継ぎ

### 3.1 引き継ぐファイル一覧

| ファイル | 用途 | 場所 |
|---------|------|------|
| `reply-mail.md` | メール処理コマンド定義 | `.claude/commands/` |
| `reply-learnings.json` | 返信作成の学習ルール | `.claude/data/` |
| `label-rules.json` | ラベル振り分けルール | `.claude/data/` |
| `label-history.json` | 送信者別ラベル履歴 | `.claude/data/` |
| `philosophy.md` | 経営フィロソフィー | `_shared/` |

### 3.2 既存ルールの維持

**メール分類ルール**（reply-mail.md より）:
- ✅ 今すぐ返信: 顧客問い合わせ、日程調整
- ⏸️ 保留: ビジネス提案、要調査案件
- ❌ 返信不要: 自動通知、広告、メルマガ

**返信作成ルール**（reply-learnings.json より）:
- L001: 日程調整は必ずカレンダー確認（正しい年で）
- L002: 資料受領済みは「拝見しました」を明記
- L003: イベント不参加は今後の日程を問い合わせ
- L004: 検討案件は関係維持の返信

**ラベル振り分けルール**（label-rules.json より）:
- 内容ベースで分類（差出人ではなく目的で判断）
- 発送通知 → 買い物
- カード利用通知 → ゴミ箱
- 日創研 → 研修関係/日創研

### 4.2 status.json の状態遷移

```

    ↓ ユーザーが「OK」と返信

    ↓ Claude Code が送信実行

needs_revision（修正待ち）
    ↓ Claude Code が修正版作成
pending（再度待機）
```

## 5. 処理フロー詳細

### 5.1 定期実行（30分ごと）

```
Claude Code 起動（cron経由）
    │
    ├─→ Phase 1: 指示処理
    │   │
    │   └─→ Google Drive の threads/ をスキャン
    │       │
    │       ├─→ status: "approved"
    │       │   └─→ メール送信 + ラベル付与 + status → "sent"
    │       │       └─→ Discord に「✅送信完了」を投稿
    │       │
    │       ├─→ status: "needs_revision" + instructions.md あり
    │       │   └─→ 修正指示を読み取り
    │       │       └─→ reply-learnings.json に学習内容追記
    │       │       └─→ draft.md を再生成
    │       │       └─→ Discord スレッドに修正版を投稿
    │       │       └─→ status → "pending"
    │       │
    │       └─→ status: "pending"（変更なし）
    │           └─→ スキップ
    │
    ├─→ Phase 2: 新着メール処理
    │   │
    │   └─→ Gmail から新着メールを取得
    │       │
    │       └─→ 各メールに対して:
    │           ├─→ 分類（既存ルール適用）
    │           ├─→ 返信案作成（reply-learnings.json 参照）
    │           ├─→ ラベル提案（label-rules.json 参照）
    │           ├─→ Google Drive に保存
    │           └─→ Discord に新規スレッド作成
    │
    └─→ Phase 3: サマリー通知
        │
        └─→ 未処理スレッドがあれば一覧を通知
```

### 5.2 Discord Bot（常駐）

```
Discord でメッセージ受信
    │
    └─→ スレッド内のメッセージか判定
        │
        ├─→ 「OK」
        │   └─→ Google Drive の status.json を "approved" に更新
        │
        ├─→ 「ラベル:〇〇」
        │   └─→ label_proposal.json を更新
        │   └─→ status.json を "approved" に更新
        │
        ├─→ 「修正：〇〇」
        │   └─→ instructions.md に内容を保存
        │   └─→ status.json を "needs_revision" に更新
        │
        ├─→ 「スキップ」
        │   └─→ status.json を "skipped" に更新
        │
        └─→ 「削除」
            └─→ trash_request フラグを追加
            └─→ status.json を "approved" に更新
```

## 6. ナレッジ蓄積の仕組み

### 6.1 修正指示からの学習

ユーザーが「修正：〇〇」と指示した場合:

1. **instructions.md に保存**
2. **Claude Code が次回起動時に読み取り**
3. **reply-learnings.json に新規ルール追加**
4. **修正版の返信案を生成**

**例: 修正指示**
```
修正：もっとフランクな文体にして
```

**例: 学習ルールへの追加**
```json
{
  "id": "L005",
  "category": "文体",
  "rule": "返信文はフランクな文体で作成する",
  "detail": "「〜いただけますでしょうか」より「〜お願いします」のようなシンプルな表現を使用",
  "added": "2026-03-16",
  "trigger": ["すべて"],
  "lesson_learned": "ユーザーからの修正指示: もっとフランクな文体にして"
}
```

### 6.2 ラベル変更からの学習

ユーザーが「ラベル:〇〇」と指示した場合:

1. **label-history.json に送信者とラベルの紐付けを保存**
2. **次回以降、同じ送信者には同じラベルを第一候補として提案**

### 8.1 既存コマンドとの共存

|---------|------|-------------|
